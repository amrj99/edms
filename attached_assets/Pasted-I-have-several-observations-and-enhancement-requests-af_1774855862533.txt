I have several observations and enhancement requests after testing. Please review and address the following:

1. Correspondence creation from Project

* Creating correspondence from inside a project currently saves it as Draft with limited fields.
* It should open the full correspondence form (To, CC, BCC, subject, attachments, priority, etc.).
* It should also automatically link the project but still allow editing before sending.
* Behavior should be consistent whether created from Project view or Correspondence module.

2. Project selector usability

* Replace long dropdowns with searchable autocomplete.
* Typing part of the project name should filter results.
* Apply the same autocomplete behavior to users, organizations, documents, etc.

3. Open correspondence in separate window

* Double-clicking a correspondence should open it in a popup or new window for easier review.

4. Attach emails to correspondence / transmittals

* Allow attaching an email (as file or reference) to correspondence.
* Allow linking existing correspondence within the system.
* Similar to Aconex-style linking between communications.

5. “Generate Link” clarification

* Confirm that this generates an external shareable link.
* If yes, ensure permissions, expiry date, and password options are available.

6. Meetings enhancements

* Option to include audio/video meeting link (Teams/Zoom/etc.)
* Field for meeting link inside meeting details.

7. Global search issue

* The global search currently returns no results; please investigate and fix.

8. Action Items clarification

* Clarify relationship between Action Items and meetings.
* Ideally Action Items should always be linked to meetings or tasks.

9. Dashboard vs Reports Dashboard

* Clarify difference or consider merging to avoid confusion.

10. Organization member assignment bug

* Adding a user to an organization sometimes does not save.
* Please verify save action and refresh state.

11. User profile improvements

* Display user's organization name.
* Add optional department field.

12. Future departments structure

* Prepare support for departments such as Finance, Contracts, Planning.
* Departments may be project-specific or organization-wide.

13. Bulk folder upload

* Ability to upload a full folder from server.
* System should scan files and classify based on type and metadata.
* Auto-place documents in correct project/location.

14. Demo project data

* Please create a full sample project including:

  * documents
  * correspondence
  * transmittals
  * meetings
  * action items
  * ITR / NCR
  * deliverables
  * tasks
    This will allow end-to-end testing.

15. Mobile application preparation

* Please confirm APIs are ready for mobile use.
* Ensure notifications and approvals can be consumed by a mobile app later.

Please confirm status for each item and implement where applicable.
