Automated Skills System — Scheduled Tasks & Smart Reminders
I want to add a configurable automation skills system. Each skill has settings that the org admin can customize. Please implement the following:
DATABASE

Add a skill_definitions table:

id, organization_id, name, description
trigger_type: enum — 'document_uploaded' | 'task_completed' | 'project_status_changed' | 'scheduled_daily' | 'scheduled_weekly' | 'scheduled_interval'
handler_type: enum — 'send_notification' | 'send_email' | 'change_status' | 'generate_report'
config (JSONB) — flexible settings per skill (recipients, intervals, thresholds, etc.)
is_enabled (boolean), created_by_id, created_at, updated_at


Add a skill_executions table:

id, skill_id, organization_id
triggered_by_type (text), triggered_by_id (integer)
status: enum — 'pending' | 'running' | 'success' | 'failed'
result (JSONB), error_message, duration_ms, executed_at
Indexes on: skill_id, organization_id, executed_at, status



BUILT-IN SKILLS (4 skills, admin-configurable per org)

Weekly Document Report (scheduled_weekly):

Config: { dayOfWeek, timeOfDay, recipients: [userId | email], projectIds? }
Action: Generate a summary of all documents per project (count by status: draft, under_review, approved, rejected). Send via email and in-app notification to configured recipients.


Unapproved Correspondence Reminder (scheduled_daily):

Config: { checkAfterHours, recipients: [userId], repeatDaily: boolean }
Action: Find all correspondence older than checkAfterHours with status not approved. Send daily reminder notification and email to configured recipients.


Technical Document Reminder (scheduled_interval):

Config: { intervalDays, documentTypes: [string], recipients: [userId] }
Action: Find all technical documents pending review for more than intervalDays. Send reminder to configured recipients. Default interval: 2 days.


Auto Project Status Change:

Trigger: task_completed event
Action: When ALL tasks in a project are completed → automatically change project status to 'completed'. When project end date has passed and status is still 'active' → change to 'on_hold' and notify project manager.



SKILL ENGINE

Create lib/skill-engine.ts:

executeSkill(skillId, context) — runs a skill and logs result to skill_executions
runScheduledSkills() — called by a cron job, finds all enabled scheduled skills and executes them
Execution is fire-and-forget with full error capture (never interrupts main flow)
Log duration_ms and status for every execution


Add a cron job in app.ts that runs runScheduledSkills() every hour and checks which scheduled skills are due.

API ENDPOINTS

Full CRUD at /api/skills (admin only):

GET /api/skills — list org skills with last execution status
POST /api/skills — create skill
PUT /api/skills/:id — update config
DELETE /api/skills/:id — delete
PUT /api/skills/:id/toggle — enable/disable
GET /api/skills/:id/executions — execution history with status and duration



All skills are org-scoped. Admin sets all config values in agreement with their team. No hardcoded defaults — everything is configurable per organization.
